# ESP32 墨水屏日历项目

本项目是一个基于ESP32的墨水屏日历设备，通过MQTT接收日历数据并显示。支持定时唤醒和低功耗设计。
日历数据来源于jisilu.cn的投资日历，由python脚本自动抓取数据并发布到MQTT服务器。

**项目参考**：根据J-Calendar项目（https://gitee.com/jadejerry/jcalendar#/jadejerry/jcalendar/blob/main/./assets/file/E-ink%20box2%20v22.3mf），在vscode完全由codebuddy修改

## 功能特性

- **双列显示**：消息以双列布局显示，充分利用墨水屏空间
- **智能日期识别**：自动识别并显示"今日"或"明日"日期前缀
- **星期显示**：根据日期自动计算并显示星期几
- **MQTT数据接收**：通过MQTT协议订阅主题，接收JSON格式的日历事件数据
- **低功耗设计**：支持深度睡眠和定时唤醒，有效降低功耗
- **状态指示灯**：LED指示灯显示设备不同工作状态（初始化、WiFi连接、MQTT连接、正常工作、AP模式、休眠准备）
- **按键控制**：
  - 短按：向MQTT服务器发送刷新请求，手动更新显示
  - 长按（2秒）：进入AP配置模式
- **Web配置界面**：在AP模式下提供友好的Web配置界面，支持：
  - WiFi网络配置（SSID、密码）
  - MQTT服务器配置（地址、端口、用户名、密码、主题）
  - 墨水屏引脚配置（CS、DC、RST、BUSY）
  - 定时唤醒时间点配置（支持最多5个时间点）
  - 单个时间点保存和删除功能
  - 配置重置功能
- **NTP时间同步**：自动从NTP服务器同步时间
- **多时间点唤醒**：支持配置多个定时唤醒时间点，系统自动选择最近的唤醒时间
- **配置持久化**：所有配置保存在ESP32的NVS中，断电不丢失
- **错误处理**：完善的错误处理和状态恢复机制

## 硬件要求

- ESP32开发板

## 使用说明

1. **上传固件到ESP32**
   - 使用PlatformIO或Arduino IDE将编译好的固件上传到ESP32开发板

2. **首次配置（AP模式）**
   - 长按按键（约2秒）进入AP配置模式
   - 设备会创建一个名为"ESP32-Config"的WiFi热点
   - 连接到该热点（无需密码）

3. **Web配置界面**
   - 在浏览器中访问 192.168.4.1
   - 配置以下信息：
     - **网络配置**：
       - WiFi SSID：您的WiFi网络名称
       - WiFi密码：您的WiFi密码
     - **MQTT配置**：
       - MQTT服务器：MQTT服务器IP地址或域名
       - MQTT端口：MQTT服务器端口（默认1883）
       - MQTT主题：用于接收日历数据的主题（如：jisilu/calendar）
       - MQTT用户名：MQTT服务器用户名（如有）
       - MQTT密码：MQTT服务器密码（如有）
     - **墨水屏引脚配置**（默认配置即可，如需修改请确认接线）：
       - CS引脚：5
       - DC引脚：17
       - RST引脚：16
       - BUSY引脚：4
     - **定时唤醒配置**：
       - 可以设置最多5个定时唤醒时间点
       - 点击"添加定时时间"按钮添加新的时间点
       - 每个时间点都有保存和删除按钮
       - 系统会在配置的时间点自动唤醒并刷新数据

4. **保存配置**
   - 点击"保存配置"按钮保存所有设置
   - 设备将自动重启并连接到配置的WiFi网络

5. **发送日历数据**
   - 通过MQTT向配置的主题发布日历数据
   - 数据格式为JSON，示例见下文"日历数据格式"

6. **查看显示**
   - 设备会在定时唤醒时自动刷新显示
   - 也可以通过短按按键手动刷新（会向MQTT发送"refresh"消息）
   - 状态指示灯会显示当前设备状态

7. **重新配置**
   - 随时可以通过长按按键进入AP模式修改配置
   - 也可以在AP模式下点击"重置所有配置"按钮清除所有设置

## 状态指示灯说明

- 快速闪烁：初始化中
- 慢速闪烁：WiFi连接中
- 双闪模式：MQTT连接中
- 常亮：正常工作
- 慢闪（1秒间隔）：AP模式
- 三次快闪后熄灭：准备进入深度睡眠

## 引脚定义

- 按键：GPIO14
- 状态LED：GPIO22
- 墨水屏：
  - BUSY：GPIO4
  - RST：GPIO16
  - DC：GPIO17
  - CS：GPIO5
  - CLK：GPIO18
  - DIN：GPIO23

## 日历数据格式

通过MQTT发送JSON格式的日历数据到配置的主题：

```json
{
  "日期": "2024-01-20",
  "data": [
    "【转债赎回】恒逸转债",
      ]
}
```

**数据格式说明**：
- `日期`：显示的日期，格式为"YYYY-MM-DD"
- `data`：事件列表数组，每个事件为一行文本
- 设备会根据日期自动计算并显示星期几
- 设备会比较当前时间，自动添加"今日"或"明日"前缀
- 支持中文字符显示
- 双列布局显示，每列最多显示约15行事件

## 注意事项

- 设备支持定时唤醒和按键唤醒两种方式
- 确保MQTT服务器可访问且消息格式正确（JSON格式）
- 墨水屏刷新频率建议不超过每小时一次，以延长屏幕寿命
- 配置信息保存在ESP32的NVS中，断电不会丢失
- 如需重置配置，可在AP模式下通过Web界面的重置按钮操作
- NTP时间同步需要在WiFi连接成功后才能进行
- 定时唤醒时间应设置在一天中的多个时间点，系统会自动选择最近的
- MQTT连接失败不影响设备的其他功能，会继续尝试重连
- 深度睡眠状态下功耗极低，只有唤醒时才会工作

## MQTT消息说明

### 订阅主题

设备订阅配置的MQTT主题，接收日历数据。

### 发布消息

短按按键时，设备会向配置的MQTT主题发布字符串"refresh"，外部系统可以根据此消息主动推送最新的日历数据。

### 数据刷新流程

1. 设备定时唤醒或短按按键
2. 向MQTT发布"refresh"消息
3. 外部系统接收到"refresh"后，推送最新日历数据
4. 设备接收并更新显示
5. 完成后进入深度睡眠

## 故障排除指南

### 1. 设备无法连接WiFi
- 检查WiFi密码是否正确
- 确保WiFi信号强度足够
- 尝试重新进入AP模式配置
- 检查WiFi频段（建议使用2.4GHz）
- 查看串口输出的调试信息

### 2. MQTT连接失败
- 验证MQTT服务器地址和端口是否正确
- 检查MQTT用户名和密码
- 确认网络防火墙是否允许MQTT通信
- 检查MQTT服务器是否正常运行
- 确认设备与MQTT服务器在同一网络或网络可达

### 3. 显示屏不更新
- 检查MQTT主题是否正确
- 验证发送的JSON数据格式是否正确
- 查看设备状态指示灯判断工作状态
- 检查墨水屏引脚连接是否正确
- 尝试短按按键手动刷新

### 4. 定时唤醒失效
- 检查NTP时间同步是否成功
- 确认定时唤醒时间配置正确
- 观察LED指示灯判断是否正常休眠
- 检查ESP32的RTC是否正常工作
- 确保没有持续的网络连接导致无法休眠

### 5. LED指示灯异常
- 快速闪烁持续：初始化失败，检查硬件连接
- 慢闪持续：WiFi连接失败，检查网络配置
- 双闪持续：MQTT连接失败，检查MQTT配置

### 6. 无法进入AP模式
- 检查按键是否正常工作
- 确认长按时间超过2秒
- 尝试重启设备后再长按按键
- 检查按键引脚连接（GPIO14）

### 7. 中文显示异常
- 检查是否使用了正确的中文字体
- 确认U8g2库已正确安装
- 查看串口输出是否有字体加载错误
